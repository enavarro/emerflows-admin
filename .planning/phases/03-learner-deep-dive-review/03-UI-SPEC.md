## Spacing Scale

Standard 8-point scale. All values must be multiples of 4px. Consistent with Phase 2 components.

| Token | px | Tailwind | Usage |
|-------|----|---------|-----------------------------------------|
| xs | 4 | `gap-1` / `p-1` | Icon-to-label gap, inline element gaps, badge padding |
| sm | 8 | `gap-2` / `p-2` | Tight element spacing, badge row gaps |
| md | 16 | `gap-4` / `p-4` | Default card content padding, between-section gaps |
| lg | 24 | `gap-6` / `p-6` | Section spacing within a page column |
| xl | 32 | `gap-8` / `p-8` | Spacing between major content blocks (player → transcript) |
| 2xl | 48 | `gap-12` / `py-12` | Empty-state vertical centering |
| 3xl | 64 | `gap-16` | Not used in Phase 3 surfaces |

**Exception:** Audio player `<audio controls>` height is browser-native (~40–54px). Do not constrain it with a fixed height utility.

**Consistent with Phase 2:** `flex flex-col gap-4` between module cards on learner page; `flex flex-col gap-6` between the three content blocks on submission viewer (player/notice → transcript → tips, or intro → Q&A → summary).

## Typography

Four roles used in Phase 3. Weights stay at two: 400 (regular) and 600 (semibold). No bold/700 — consistent with existing Phase 2 components.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Display | 20px | `text-xl` | 600 semibold | 1.25 (`leading-tight`) | Learner name as `pageTitle` heading; module card titles (`M01 · Title`) |
| Body | 16px | `text-base` | 400 regular | 1.5 (`leading-relaxed`) | Transcript line text; Q&A answer text; polished intro quote; exercise summary text; tip prose |
| Label | 14px | `text-sm` | 400 regular | 1.4 (`leading-snug`) | Badge text; metadata (attempt, date, module label); question labels; category labels above tip groups |
| Caption | 12px | `text-xs` | 400 regular | 1.3 (`leading-none`) | Mono external_id label; upper-tracking section labels (`uppercase tracking-wider`); pronunciation tooltip text |

**Specific decisions:**

- Learner name: `text-xl font-semibold text-brand-teal` (matches cohort-card h3 pattern from Phase 2)
- Module card title: `text-base font-semibold text-brand-teal`
- Transcript line text: `text-base text-foreground leading-relaxed`
- Q&A question label: `text-sm font-semibold text-muted-foreground uppercase tracking-wider`
- Q&A answer text: `text-base text-foreground`
- Polished intro quote: `text-base italic text-brand-cream leading-relaxed` (on dark teal card)
- Tip category label: `text-xs uppercase tracking-wider text-muted-foreground` (above grouped tip cards)
- Tip body: `text-sm text-foreground`
- External ID: `font-mono text-xs text-muted-foreground`

## Surface Specifications

### Surface 1: Learner Detail Page

**Route:** `/dashboard/teach/cohorts/[cohort]/learners/[learnerId]`

**Page header (PageContainer):**
- `pageTitle`: learner's name (e.g., `"Ana García"`)
- `pageDescription`: `"{cohort} · {level || '—'}"` — external_id appended as ` · {externalId}` when non-null
- `pageHeaderAction`: none on this page

**External ID display:** Below the `PageContainer` description, render a `<p className='font-mono text-xs text-muted-foreground mt-1'>` with the external_id value. Only render when non-null. This is supplemental to the description, not part of `pageDescription` itself (description has limited line wrapping).

**Profile summary section:** PageContainer header handles the name + cohort/level description. No additional summary card needed. Consistent with D-09.

**Module card layout (D-05, D-08):**

```
<div className='flex flex-col gap-4'>
  {/* One Card per module with submissions, sorted by module number ascending */}
  <Card>
    <CardHeader>
      <h2>M01 · Introduce Yourself</h2>
    </CardHeader>
    <CardContent>
      {/* Submission rows, newest-attempt-first */}
    </CardContent>
  </Card>
</div>
```

**Module card title:** `text-base font-semibold text-brand-teal` — format: `M{NN} · {ModuleDef.title}` (zero-padded two-digit module number)

**Submission row structure (inside each card):**

Row is a clickable `<Link>` (stretched-link pattern from `learners-table.tsx` — `after:absolute after:inset-0`):

```
[Type badge] [Att N badge] [Status badge] [Reviewed/Pending indicator] [Date] →
```

- Type badge: `<Badge variant='outline'>Recording</Badge>` or `<Badge variant='outline'>Conversation</Badge>`
- Attempt badge: `<Badge variant='secondary'>Att 1</Badge>` (attempt number from `attemptNum`)
- Status badge: `<Badge variant='outline'>{status}</Badge>` — plain text from `submission.status`
- Reviewed indicator: when `reviewedAt` is non-null → `<Badge className='bg-brand-sage/20 text-brand-teal border-brand-sage'>Reviewed</Badge>`; when null → `<Badge variant='outline' className='text-muted-foreground'>Needs review</Badge>`
- Date: `text-sm text-muted-foreground` — formatted as `MMM d, yyyy` (same pattern as `progress-matrix.tsx`)

**Row separator:** `<Separator />` between rows within a card (not after the last row)

**Empty state (no submissions for learner):**

```
<div className='flex flex-col items-center justify-center gap-3 rounded-md border bg-card py-12 text-center'>
  <Icons.clipboardList className='h-8 w-8 text-muted-foreground' aria-hidden='true' />
  <h2 className='text-lg font-semibold text-brand-teal'>No submissions yet</h2>
  <p className='max-w-sm text-sm text-muted-foreground'>
    This learner has not submitted any work yet.
  </p>
</div>
```

**Skeleton fallback (Suspense):**

```
<div className='flex flex-1 animate-pulse flex-col gap-4 p-4 md:px-6'>
  <div className='bg-muted mb-2 h-8 w-48 rounded' />
  <div className='bg-muted h-4 w-72 rounded' />
  <div className='bg-muted mt-4 h-32 w-full rounded-lg' />
  <div className='bg-muted h-32 w-full rounded-lg' />
</div>
```

```
<Card>
  <CardHeader>
    <h3 className='text-sm font-semibold text-muted-foreground uppercase tracking-wider'>
      Transcript
    </h3>
  </CardHeader>
  <CardContent>
    <div className='flex flex-col gap-3'>
      {/* One TranscriptLine per RecordingTranscriptEntry */}
    </div>

**Transcript line structure (`TranscriptLine` component):**

- Outer: `<div className='border-l-2 {classificationBorderClass} pl-3 py-0.5'>`
- Classification → border class:
  - `strong` → `border-brand-sage`
  - `needs improvement` → `border-amber-500`
- Inner text: words rendered individually. Words with a non-empty `pronunciation` field in `words[]` array:
  - Render as `<Tooltip>` trigger with `underline decoration-dotted decoration-brand-sage decoration-1 cursor-default` (subtle sage dotted underline)
  - `<TooltipContent>` shows pronunciation hint text (e.g., `IPA: /wɜːd/` or raw `pronunciation` field value)
- Words without hints: plain `<span>` with no decoration
- Words joined by spaces (reconstruct the sentence)

Flat vertical list, grouped by category. Category headers above each group.

```

### Surface 4: Mark-Reviewed Control

Rendered in `pageHeaderAction` slot on both submission viewer types (REV-01..REV-04).

**Three button states:**

**State: idle (not yet reviewed)**

```
<Button
  onClick={handleMarkReviewed}
  disabled={isPending}
  className='bg-brand-teal text-white hover:bg-brand-teal/90'
>
  {isPending
    ? <><Icons.loader className='mr-2 h-4 w-4 animate-spin' aria-hidden='true' />Saving…</>
    : <><Icons.check className='mr-2 h-4 w-4' aria-hidden='true' />Mark as reviewed</>
  }
</Button>
```

Label copy: `"Mark as reviewed"` (idle) / `"Saving…"` (pending)
Icon: `Icons.check` (idle), `Icons.loader` (pending, with `animate-spin`)
Style: `bg-brand-teal text-white` — teal CTA matching brand conventions

**State: reviewed**

Replace button with an inline group (not a `<Button>`):

```
<div className='flex items-center gap-3'>
  <span className='flex items-center gap-1.5 text-sm text-muted-foreground'>
    <Icons.check className='h-4 w-4 text-brand-sage' aria-hidden='true' />
    Reviewed by instructor on {formattedDate}
  </span>
  <Button
    variant='ghost'
    size='sm'
    onClick={handleUndo}
    disabled={isPending}
  >
    {isPending
      ? <Icons.loader className='h-3 w-3 animate-spin' aria-hidden='true' />
      : <><Icons.undo className='mr-1 h-3 w-3' aria-hidden='true' />Undo</>
    }
  </Button>
</div>
```

`formattedDate`: `format(new Date(reviewedAt), 'MMM d, yyyy')` — same date format as progress-matrix.
Reviewer label: `"Reviewed by instructor on {date}"` — generic, no profiles JOIN (D-06).
Undo label: `"Undo"` with `Icons.undo` — no confirmation dialog (D-07).
Undo pending label: spinner only (no text, small `size='sm'`).

**Toast messages (sonner):**

| Event | Toast |
|-------|-------|
| Mark reviewed success | `"Submission marked as reviewed"` |
| Mark reviewed error | `error.message` (from API response) |
| Undo success | `"Review undone"` |
| Undo error | `error.message` |
