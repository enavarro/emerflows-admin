## Spacing Scale

Declared values (must be multiples of 4) — uses Tailwind v4 default `4px` step. Maps cleanly to shadcn primitives (which already use multiples of 4) and the brand `rounded-[2rem]` mobile / `rounded-[3rem]` desktop convention from CLAUDE.md.

| Token | Value | Usage in Phase 2 |
|-------|-------|------------------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-text gap inside chevron affordance, dot-to-label gap inside status pill |
| sm | 8px (`gap-2`, `p-2`) | Card stat-grid inner gaps, badge horizontal padding, tooltip padding |
| md | 16px (`gap-4`, `p-4`) | Default cohort-card padding, tab content top padding, learner-table cell padding |
| lg | 24px (`gap-6`, `p-6`) | Cohort grid gap (between cards), section padding around the matrix table |
| xl | 32px (`gap-8`, `py-8`) | Vertical breathing room between page header and first card row |
| 2xl | 48px (`py-12`) | Empty-state vertical padding (centered cohort/learner zero-state blocks) |
| 3xl | 64px (`py-16`) | Reserved — not used in Phase 2 (page-level spacing handled by `PageContainer`) |

**Cohort card geometry:**
- Card outer corner radius: `rounded-2xl` (16px) — matches existing `Card` primitive default; preserves consistency with `src/features/demos/` and `src/features/users/` cards. (CLAUDE.md `rounded-[2rem]/[3rem]` rule applies to learner-facing brand surfaces, not admin tables — admin keeps the standard shadcn radius for visual coherence with the rest of the dashboard.)
- Card minimum height: ~220px (mirrors prototype `CohortCard`); flex layout so contents distribute naturally.
- Cohort grid: CSS grid `repeat(auto-fill, minmax(340px, 1fr))` with `gap-6` (24px). Mirrors prototype IA.

**Matrix cell geometry:**
- Cell size: 32×32px (`h-8 w-8`) — comfortable hover target without dominating the row
- Dot diameter: 12px (`h-3 w-3`) — three encodings render at the same size, only fill/border differ (D-16)
- Dot border thickness: 1.5px for `not-started` and `submitted` rings (visually distinct from filled `reviewed` dot)
- Sticky header row offset: top-0 inside the matrix scroll container
- Sticky learner-name column offset: left-0 inside the matrix scroll container; min-width 200px

**Exceptions:** none — all spacing resolves to Tailwind 4-step multiples.

## Typography

Project uses Tailwind v4 default type scale with the Geist font (from `src/app/layout.tsx`). Phase 2 declares the four roles below. All sizes are multiples of 4 except `text-xs` (12px) and `text-sm` (14px) which are the project's existing UI-text defaults inherited from shadcn — kept for cross-feature consistency with `src/features/demos/` and `src/features/users/`.

| Role | Size | Weight | Line Height | Used for (Phase 2) |
|------|------|--------|-------------|--------------------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 (`leading-6`) | Card stat values, learner table cells, tooltip text, helper copy |
| Label | 12px (`text-xs`) | 600 (`font-semibold`) + `uppercase tracking-wider` | 1.4 (`leading-tight`) | Cohort-card stat labels ("STUDENTS", "COMPLETION", "NEEDS REVIEW", "NEXT"), table column headers, status pill text |
| Heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 (`leading-7`) | Cohort name on each card |
| Display | 28px (`text-3xl` ≈ 30px in Tailwind defaults; locked to **28px** here for exact 4-multiple) | 700 (`font-bold`) | 1.2 (`leading-tight`) | Page title in `PageContainer` header — "Cohorts" (list) and "{Cohort name}" (detail) |

**Resolution note for executor:** The `Heading` component used by `PageContainer` already applies a default heading style; do NOT override unless visual fidelity demands it. If a manual size is needed for the page title, use `text-3xl font-bold` (Tailwind's 30px default — within tolerance of the declared 28px display token).

**Exactly 3 weights are in active use** (400 body, 600 label/heading, 700 display) — the project's existing pattern. This is one weight above the gsd preferred 2-weight target; justified because the existing `Heading` component already uses bold at the page level, and re-implementing it would violate `Page headers — use PageContainer ... never import <Heading> manually` (CLAUDE.md).

**Open question (defaulted):** No display vs heading separation in matrix table cells — module column labels render as Body weight 600 in `tabular-nums` for alignment.

| Accent (10%) | `brand-sage` `#78F995` | Reviewed-state matrix dot fill, "Active" status pill dot, primary semantic "good/done" affordances |

| Ink — muted | Tailwind `text-muted-foreground` (zinc-500ish) | Stat labels (uppercase), term hint, "Reviewed" / "Needs review" subtext, table secondary cells |
| Border — soft | `brand-cream` `#F5F0E8` or `border-zinc-200` | Card borders, table row dividers, matrix `not-started` cell hairline ring |

**Accent reserved for (10% rule — explicit list):**

1. Filled dot in matrix cells where `state === 'reviewed'` (D-16, D-05) — `bg-brand-sage`
2. Outline ring in matrix cells where `state === 'submitted'` (D-16) — `border-brand-sage` ring at 1.5px, transparent fill
3. Status pill dot for `Active` cohort status (D-07 — single status today) — 6px `bg-brand-sage` circle inside the pill

**Accent is NOT used for:** "+ New cohort" button (it is disabled — uses `Button` `variant='outline'` muted styling), "Export" button (same), tab active state (uses `brand-teal` ink, not accent), card hover (uses `brand-cream` background lift only), chevron affordance on cards (uses `brand-teal` ink at the bottom-right `Open →` cue).

**Cell encoding spec (D-16, sage-progressive — locked):**

| State | Visual | Tailwind classes |
|-------|--------|------------------|
| `not-started` | Empty hairline ring, no fill | `h-3 w-3 rounded-full border border-brand-cream bg-transparent` |
| `submitted` | Sage outline ring, no fill | `h-3 w-3 rounded-full border-[1.5px] border-brand-sage bg-transparent` |

No amber, no rose, no destructive color in matrix cells — the prototype's amber/rose are reserved for transcript classification (Phase 3) per CLAUDE.md transcript color rules.

### Date / relative-time formatting

- **Tooltip dates** (matrix cells): `Apr 22, 2026` — absolute, short month, four-digit year. Use `date-fns` `format(date, 'MMM d, yyyy')`. Project already depends on `date-fns 4.1.0`.
- **Latest activity column** (learner table): `2h ago`, `Yesterday`, `3 days ago`, then absolute `Apr 22, 2026` for >7 days. Use `date-fns` `formatDistanceToNowStrict` with custom thresholds; OR fallback to `format(date, 'MMM d')` for >30 days. Discretion — executor picks the cleanest implementation.
- **Card term hint**: static string per D-08, no formatting needed in v1.

### Disabled-button accessibility

Both "+ New cohort" and "Export" placeholders MUST set:
- `disabled` attribute on the underlying `<button>`
- `aria-disabled='true'`
- `title='Coming soon'` for native tooltip on hover (sufficient — no `Tooltip` primitive needed)

This avoids spurious click handlers and screen-reader confusion while preserving prototype IA.

## Interaction Contract

Phase 2 has very few interactive surfaces (it's a read-only browse). All interactions:

| Surface | Trigger | Behavior | Accessibility |
|---------|---------|----------|---------------|
| Cohort card | Click anywhere on card | Navigate via `Link` to `/dashboard/teach/cohorts/{cohort.id}` | Whole card wrapped in `<Link>`; `aria-label='Open {cohort.name}'`; visible focus ring inherited from `Card` primitive |
| Cohort card | Hover | Background lifts to `brand-cream`, optional `transform translate-y-[-2px]` and soft shadow (mirror prototype IA, but tone-down — admin app, not marketing) | No keyboard equivalent needed — focus ring serves |
| Disabled placeholder buttons | Click | No-op — `disabled` prevents click | `aria-disabled='true'`, `title='Coming soon'` |
| Tabs (Learners / Progress matrix) | Click on `TabsTrigger` | Switch active tab; default = `Learners` (D-13) | Inherited from shadcn `Tabs` (radix-ui) — full keyboard nav (←→ arrow keys, Home/End) |
| Learner table row | Click anywhere on row | Navigate to `/dashboard/teach/cohorts/{cohort.id}/learners/{learner.id}` (URL only — page wired in Phase 3) | Whole row inside `<Link>`; row has `cursor-pointer` and `brand-cream` hover; visible focus ring |
| Matrix module-header (M01..M12) | Hover | Show shadcn `Tooltip` with full module title | `aria-label` on header `<th>` includes full module title |
| Matrix cell | Hover | Show shadcn `Tooltip` with state-specific copy (D-18) | `aria-label` on each cell summarises state for screen readers (e.g. `Spring 2026 — Alice — Module 1 — Reviewed Apr 22, 2026`); cell is **not** focusable / interactive in Phase 2 (D-17 — no navigation; that lands in Phase 3+) |

**Loading orchestration:**
- Both pages use the Phase 1 RSC pattern: server component calls `void queryClient.prefetchQuery(cohortsQueryOptions())` (or `cohortQueryOptions(cohortId)`) and wraps the client component in `<HydrationBoundary state={dehydrate(queryClient)}>`.
- Each major surface (cohorts grid; learner table; progress matrix) is wrapped in its own `<Suspense fallback={<Skeleton .../>}>` for streaming.
- Reference pattern: `src/features/demos/components/demos-listing.tsx`.

**Error orchestration:**
- Each route gets a sibling `error.tsx` in its app-router segment that renders the error state copy above. Reference: any existing `error.tsx` in `src/app/dashboard/*/`.

## Open Questions (defaulted, executor may flag)

1. **Cohort card click-target vs inner buttons** — Phase 2 has no inner buttons on cohort cards; entire card is the link. Defaulted: whole card click-through. If a future phase adds an inner action (e.g. "Quick view"), the card must split into outer link + inner button-stop-propagation.
2. **Matrix horizontal scroll on narrow viewports** — Defaulted: `ScrollArea` wraps the matrix `<table>`; sticky learner-name column keeps context when scrolling. Mobile-narrow behavior is acceptable per "Mobile-optimized layout — out of scope" (REQUIREMENTS.md).
3. **Matrix legend placement** — Defaulted: rendered below the matrix table, left-aligned, three swatches inline. If user prefers above the table or none at all, executor may move/remove.
4. **`Icons.plus` availability** — Sample of icon registry shows `Icons.check`, `Icons.dots`, etc. but not explicit `Icons.plus`. Defaulted: render the `+` glyph as a literal text character inside the disabled "+ New cohort" button (no icon needed); if a `plus` icon exists or is added to the registry, use it.
5. **Card hover lift magnitude** — Prototype uses `transform translateY(-2px)` + soft shadow. Defaulted: keep lift but use Tailwind `hover:-translate-y-0.5 hover:shadow-md transition` (admin app, gentler lift than prototype).
